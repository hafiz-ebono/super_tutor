# Roadmap: Super Tutor

## Overview

Super Tutor is built in three natural delivery phases. Phase 1 delivers the complete end-to-end loop: a user submits a URL, picks a tutoring type, and arrives at a working tabbed study page with notes, flashcards, and a quiz — all generated by Agno-powered agents with a configurable AI provider. Phase 2 adds the second, more complex ingestion path: topic description with AI-driven web research. Phase 3 completes the study experience with interactive flashcard flip, turning a functional product into a polished one. Each phase produces something runnable and verifiable before the next begins.

## Phase Details

### Phase 1: URL Session Pipeline
**Goal**: Users can create a complete study session from a URL and immediately study the material
**Depends on**: Nothing (first phase)
**Requirements**: SESS-01, SESS-03, SESS-04, SESS-05, GEN-01, GEN-02, GEN-03, STUDY-01, AGENT-01, AGENT-02
**Success Criteria** (what must be TRUE):
  1. User can submit an article/doc URL with a focus prompt and a tutoring type, and receive a tabbed study page with notes, flashcards, and a quiz within one request
  2. User sees step-by-step progress messages during generation (extracting, generating notes, creating flashcards, building quiz) — the page is never silently waiting
  3. When a URL fails (paywall, empty page, invalid), user sees a specific error message and is offered a text-paste fallback to continue without losing their session
  4. Notes, flashcards, and quiz are all observably different in tone and complexity across the three tutoring types (Micro Learning, Teaching a Kid, Advanced) on the same input
  5. Switching the AI provider or model requires only a config/environment change — no code changes are needed
