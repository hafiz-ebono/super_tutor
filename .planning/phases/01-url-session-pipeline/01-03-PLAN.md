  <behavior>
    Chain logic (extract_content):
    1. If jina_api_key is empty → skip Jina layer entirely → try trafilatura
    2. Try Jina Reader: GET https://r.jina.ai/{url} with Bearer token, 15s timeout
       - If response text len > 200 chars → return text (success)
       - If exception or short response → fall through to layer 2
    3. Try trafilatura: fetch_url(url) → extract() with include_tables=True, output_format="markdown"
       - trafilatura.extract() returns None on failure — always check
       - If text len > 200 chars → return text (success)
       - Otherwise → fall through to layer 3
    4. Try Playwright: launch chromium, goto url (20s timeout), wait networkidle, page.content()
       - Pass raw HTML through trafilatura.extract()
       - If text len > 200 chars → return text (success)
       - Otherwise → fall through to error
    5. All layers failed → raise ExtractionError(kind=_classify_failure(url))

    Error classification (_classify_failure):
    - "invalid_url": URL does not start with http:// or https://
    - "paywall": URL domain matches a known paywall list (nytimes.com, wsj.com, ft.com, bloomberg.com, economist.com)
    - "empty": all other failures (default)
    - "unreachable": for future use; not auto-classified in Phase 1

    Test cases (input → expected output):
    - Jina returns >200 chars → returns that text (no fallback)
    - Jina returns short text + trafilatura returns >200 chars → returns trafilatura text
    - Jina fails exception + trafilatura returns None + playwright returns >200 chars → returns playwright text
    - All three fail → raises ExtractionError
    - ExtractionError.kind == "invalid_url" when URL has no scheme
    - ExtractionError.kind == "paywall" when URL is nytimes.com
    - ExtractionError.kind == "empty" for unknown domain that fails
    - jina_api_key empty → Jina layer never called
  </behavior>

    **Individual extractor functions:**

    backend/app/extraction/jina.py:
    ```python
    import httpx

    async def fetch_via_jina(url: str, api_key: str) -> str | None:
        """Returns text if successful, None if failed or content too short."""
        try:
            async with httpx.AsyncClient(timeout=15) as client:
                resp = await client.get(
                    f"https://r.jina.ai/{url}",
                    headers={
                        "Authorization": f"Bearer {api_key}",
                        "X-Return-Format": "text",
                    },
                )
                resp.raise_for_status()
            text = resp.text.strip()
            return text if len(text) > 200 else None
        except Exception:
            return None
    ```

    backend/app/extraction/trafilatura_extractor.py:
    ```python
    import trafilatura

    def fetch_via_trafilatura(url: str) -> str | None:
        """Returns extracted markdown text or None."""
        downloaded = trafilatura.fetch_url(url)
        if not downloaded:
            return None
        text = trafilatura.extract(
            downloaded,
            include_tables=True,
            no_fallback=False,
            output_format="markdown",
        )
        return text if text and len(text) > 200 else None

    def extract_from_html(html: str) -> str | None:
        """Used by Playwright layer to extract from already-fetched HTML."""
        text = trafilatura.extract(
            html,
            include_tables=True,
            output_format="markdown",
        )
        return text if text and len(text) > 200 else None
    ```

    backend/app/extraction/playwright_extractor.py:
    ```python
    from app.extraction.trafilatura_extractor import extract_from_html

    async def fetch_via_playwright(url: str) -> str | None:
        """Launches headless Chromium, returns extracted text or None."""
        try:
            from playwright.async_api import async_playwright
            async with async_playwright() as p:
                browser = await p.chromium.launch()
                page = await browser.new_page()
                await page.goto(url, timeout=20000)
                await page.wait_for_load_state("networkidle", timeout=15000)
                html = await page.content()
                await browser.close()
            return extract_from_html(html)
        except Exception:
            return None
    ```

    backend/app/extraction/chain.py:
    ```python
    from app.config import get_settings
    from app.extraction.jina import fetch_via_jina
    from app.extraction.trafilatura_extractor import fetch_via_trafilatura
    from app.extraction.playwright_extractor import fetch_via_playwright

    class ExtractionError(Exception):
        def __init__(self, kind: str, message: str = ""):
            self.kind = kind
            self.message = message
            super().__init__(message or kind)

    PAYWALL_DOMAINS = ["nytimes.com", "wsj.com", "ft.com", "bloomberg.com", "economist.com"]

    def _classify_failure(url: str) -> str:
        if not url.startswith(("http://", "https://")):
            return "invalid_url"
        if any(d in url for d in PAYWALL_DOMAINS):
            return "paywall"
        return "empty"

    async def extract_content(url: str) -> str:
        """
        Tries Jina → trafilatura → Playwright. Returns text or raises ExtractionError.
        Skips Jina if JINA_API_KEY is not configured.
        """
        settings = get_settings()

        # Layer 1: Jina Reader (skip if no API key)
        if settings.jina_api_key:
            text = await fetch_via_jina(url, settings.jina_api_key)
            if text:
                return text

        # Layer 2: trafilatura (sync, run directly — not blocking for short I/O)
        text = fetch_via_trafilatura(url)
        if text:
            return text

        # Layer 3: Playwright (async headless browser)
        text = await fetch_via_playwright(url)
        if text:
            return text

        raise ExtractionError(
            kind=_classify_failure(url),
            message="Could not extract readable content from this URL",
        )
    ```

    Note: trafilatura.fetch_url() is synchronous and performs HTTP I/O. For Phase 1, running it directly in the async event loop is acceptable since Playwright (the heavy layer) is already async. If performance becomes an issue, wrap in asyncio.to_thread().
