    **Test 1 — Full URL flow (primary path):**
    1. Open http://localhost:3000
    2. Click the CTA button — confirm it goes to /create
    3. Select "Micro Learning" mode
    4. Enter a real article URL (e.g. https://en.wikipedia.org/wiki/Machine_learning)
    5. Enter a focus prompt: "key algorithms"
    6. Click "Generate my study session →"
    7. Confirm the loading page appears full-screen with progress bar at top
    8. Watch progress bar advance as each step message changes ("Reading the article...", "Crafting your notes...", etc.)
    9. After auto-redirect to /study/{id}:
       - Confirm sidebar shows article title and "Micro Learning" label
       - Click "Notes" — confirm structured markdown with headings and bullets renders
       - Click "Flashcards" — confirm grid of question-side cards (no flip)
       - Click "Quiz" — answer each question, confirm instant correct/wrong color feedback
       - After last question: confirm "X/Y correct" score and full question review
       - Confirm "Retake quiz" button resets the quiz

    **Test 2 — Tutoring type comparison (SESS-03):**
    1. Go to /create, select "Advanced" mode
    2. Submit the SAME URL as Test 1 with the SAME focus prompt
    3. On the resulting study page, compare Notes with Test 1 — confirm "Advanced" notes are visibly more technical, use jargon, more complex sentences
    4. Compare flashcard question complexity — "Advanced" should ask more nuanced questions

    **Test 3 — URL failure + paste fallback (SESS-04):**
    1. Go to /create, select any mode
    2. Enter an invalid URL: "not-a-url"
    3. Submit — confirm error appears inline below the URL field with friendly message ("We couldn't read that page") and a specific pointer
    4. Confirm the paste textarea appears
    5. Confirm mode selection and focus prompt are still filled in
    6. Paste several paragraphs of text (>200 chars)
    7. Submit — confirm the same loading screen appears and produces a complete session

    **Test 4 — Provider config (AGENT-02):**
    1. Check which provider is configured in backend/.env
    2. Note the generation style/model behavior
    3. (Optional, if another provider key is available) Switch AGENT_PROVIDER and AGENT_MODEL in .env, restart backend server, generate another session — confirm different model behavior without any code changes

    **Pass criteria:**
    - All three panels (Notes, Flashcards, Quiz) are functional with real AI-generated content
    - Progress bar visibly advances step-by-step (not all at once at the end)
    - "Micro Learning" and "Advanced" sessions on same URL are observably different in tone and complexity
    - URL failure shows specific error + paste fallback; paste path produces complete session
    - "New session" link in sidebar navigates to /create
