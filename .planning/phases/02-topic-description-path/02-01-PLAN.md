  <name>Task 1: Create research_agent.py with Agno + DuckDuckGoTools</name>
  <files>backend/app/agents/research_agent.py</files>
  <action>
Create `backend/app/agents/research_agent.py` with:

1. A `ResearchResult` dataclass with two fields:
   - `content: str` — synthesized research text ready to feed into the notes/flashcard/quiz agents
   - `sources: list[str]` — list of source URLs used (3–5 URLs)

2. A `build_research_agent()` factory function that returns an Agno `Agent` configured with:
   - `model=get_model()` (reuse existing model factory — per AGENT-01 pattern, all agents use same configurable model)
   - `tools=[DuckDuckGoTools(enable_search=True, enable_news=False, fixed_max_results=5)]`
   - `instructions` system prompt telling the agent to: search for the topic using 2-3 targeted queries, synthesize findings into a comprehensive content body (at least 600 words), list the source URLs, and return a JSON object with exactly two keys: `"content"` (the synthesized text) and `"sources"` (array of URL strings). Include "Return ONLY valid JSON, no markdown fences, no explanation" to prevent LLM wrapping output in prose (same pattern as flashcard/quiz agents).

3. A `run_research(topic: str, focus_prompt: str = "") -> ResearchResult` function that:
   - Builds the agent via `build_research_agent()`
   - Constructs an input string: if focus_prompt provided, append "Focus on: {focus_prompt}" to the topic
   - Calls `agent.run(input)` and parses the JSON result using `_parse_json_safe` logic (strip markdown fences with regex, parse JSON, fallback if malformed)
   - Returns a `ResearchResult` with content and sources
   - On any exception, returns `ResearchResult(content="", sources=[])` (caller handles empty content as research failure)

Import pattern:
```python
from agno.agent import Agent
from agno.tools.duckduckgo import DuckDuckGoTools
from app.agents.model_factory import get_model
```

Note: DuckDuckGoTools uses the `ddgs` package (no API key required — free web search). If `ddgs` is not installed, add it to requirements.txt: `ddgs>=6.0.0`.

Check if `ddgs` is already installed: `pip show ddgs`. If missing, add to `backend/requirements.txt` and run `pip install ddgs` in the backend venv.
